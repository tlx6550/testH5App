此demo出现以下问题
1.UI 按1080设计尺寸，H5嵌入某个WebView中
如果计算根大小的flexlabe.js 传入的参数是1080，那么导致的轮播图高度不会吻合高保的设计，但是单独打开这个h5页面确实符合的；

解决方案：以嵌入客户端的WebView实际要求的宽度为准，这里高保的WebView宽度是1008，所以参数是1008即可；

2.swiper 轮播通过ifram标签嵌入第三方页面的时候，当滑到第三方页面的时候无法滑动了，原因是第三方的页面无法监听滑动的事件动作了，

解决方案，把滑动页面的需求移交给客户端管理实现！

3.轮播图和上方的切换冲突，当滑到轮播图的时候，切换也同时切换了！

解决方案：客户端通过暴露方法解决
function  addSwiperUrls(urls)
{
  return window.mmutil.addSwiperUrls(urls);
}
var urlArr = ['轮播图片地址']
addSwiperUrls(urlArr )

4.页面下载应用，在某些安卓低版本系统无法下载，原因是H5页面引入了某个使用es6语法写的js，导致客户端加载时候，无法编译通过（即使引入了babel-polyfill.js编译），导致js挂起无法下载；
解决方案：
单独对含es6语法的js编译编译成es5语法后再引入，成功解决！
（在线转换https://babeljs.io/）
